Etiquetas HTML

  • DOCTYPE:

    Define el tipo de documento con el que estamos trabajando. Comando ! + enter.

  • HTML:

    Define el tipo de documento.

  • HEAD:

    Información básica del documento (metadatos, hojas de estilo para css, etc.).

  • HEADER:

    Define la cabecera del proyecto. Usualmente contiene el nombre del sitio, además de la barra de navegación principal.

  • BODY:

    Representa el contenido básico del proyecto.

  • MAIN:

    Contiene la información más relevante e importante de todo el proyecto. SOLO PUEDE HABER UN MAIN POR TRABAJO.

  • FOOTER:

    Es el pie de página del proyecto, aquí va la información de contacto o una firma.

  • SECTION:

    Define, como su nombre lo señala, una sección del proyecto.

  • ARTICLE:

    Contenido dentro de la sección que le corresponda. Es material independiente, representa su propia información.

  • H1-H2-H3-H4-H5-H6:

    Elementos título para las cabeceras de cada texto o párrafo. Mientras más alto el número, menos es su jerarquía, tamaño y relevancia. NO ES BUENA IDEA TENER MÁS DE UN H1, deben de preservarse para títulos importantes.

  • P:

    Convierte las líneas de texto en un párrafo.

  • BR:

    Es un salto de línea.

  • UL:

    Define una lista que no necesariamente obedece un orden.

  • LI:

    Define los artículos enumerados en una lista, y está contenido dentro de UL.

  • DIV:

    Es un contenedor genérico, y a diferencia de SECTION no tiene elementos relacionados a un tema específico dentro de sí. DIV NO DEFINE SECCIONES, no tiene una jerarquía real.

  • A:

    Es para introducir un hyperenlace.

  • Q:

    Representa una cita textual.

  • IMG:

    Representa literalmente a una imagen, trabaja usualmente con A.

  • BUTTON:

    Es para colocar un botón en el proyecto.

  • NAV:

    Define el área para el menú de navegación.
